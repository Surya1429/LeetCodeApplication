🧠 What is application.properties?

Mobile App → Settings (WiFi, Theme, Notifications)

🔥 Why We Use It?
👉 Instead of writing values in code, we keep them here:

Without properties	With properties
Hardcoded ❌	Configurable ✅
Difficult to change ❌	Easy to change ✅

🔹 Write This Code
package com.leetcode.authservice.entity;

import jakarta.persistence.*;
import lombok.*;

@Entity
@Table(name = "users")
@Data
@NoArgsConstructor
@AllArgsConstructor
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String username;

    @Column(unique = true, nullable = false)
    private String email;

    @Column(nullable = false)
    private String password;

    private String role;
}
🧠 Explanation (Simple)
Annotation	Meaning
@Entity	This class = DB table
@Table	Table name = users
@Id	Primary key
@GeneratedValue	Auto increment
@Column	Column rules

🔐 What is HttpSecurity?

👉 HttpSecurity is a Spring Security class used to configure security rules for HTTP requests.

🧠 Simple Meaning
HttpSecurity = controls who can access which API
🎯 What It Does

Using HttpSecurity, you can:

✅ Allow or block APIs
✅ Define authentication rules
✅ Disable CSRF (for Postman/testing)
✅ Add login/logout rules
🔧 Example (Your Project)
http
  .csrf(csrf -> csrf.disable())
  .authorizeHttpRequests(auth -> auth
      .requestMatchers("/auth/**").permitAll()
      .anyRequest().authenticated()
  );
🧠 Breakdown
🔹 .csrf().disable()

👉 Turns off CSRF protection (needed for testing APIs)

🔹 .requestMatchers("/auth/**").permitAll()

👉 Allows these APIs without login

🔹 .anyRequest().authenticated()

👉 All other APIs require authentication

⚡ One-Line Memory Trick
HttpSecurity = rules for securing your APIs
